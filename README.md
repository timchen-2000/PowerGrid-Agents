# 电力设备监控智能问答系统

## 项目设计目的

基于Langchain框架构建电力设备监控智能问答系统，负责Agent智能助手核心功能的设计与实现，为电力设备监控领域提供专业的智能问答服务。

## 项目描述

该系统集成了向量化文档存储与智能检索模块，基于FAISS向量数据库实现电力设备监控知识的高效检索，同时构建了电力设备监控专用工具集，实现智能工具调用和多轮对话能力。

## 主要功能

### 1. 优化RAG检索架构
- 集成向量化文档存储与智能检索模块
- 基于FAISS向量数据库实现电力设备监控知识的高效检索
- 支持PDF文档的自动加载、分割和向量化

### 2. 完成Agent工具链设计
- 构建电力设备监控专用工具集
- 包括设备状态检查、告警查询、维护计划管理等核心工具
- 实现智能工具调用和多轮对话能力

## 核心模块说明

### 1. 文档处理模块 (`document_processor.py`)
- 负责PDF文档的加载和预处理
- 实现文档的智能分割，生成适合检索的文本片段
- 支持自定义分割参数（chunk_size、chunk_overlap）

### 2. 向量存储模块 (`vector_store.py`)
- 基于FAISS构建高效的向量数据库
- 支持文档向量化、存储、加载和检索
- 提供相似度搜索和检索器功能

### 3. 智能问答Agent (`qa_agent.py`)
- 基于Langchain构建专业的电力设备监控问答系统
- 集成OpenAI模型，提供专业、准确的回答
- 支持多轮对话和上下文理解

### 4. 主程序 (`main.py`)
- 提供交互式问答界面
- 自动处理文档和构建向量数据库
- 支持系统初始化和配置

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 配置环境变量

复制 `.env.example` 文件为 `.env`，并填入你的OpenAI API密钥：

```
OPENAI_API_KEY=your_openai_api_key_here
OPENAI_API_BASE=https://api.openai.com/v1
```

### 3. 运行系统

```bash
python main.py
```

系统会自动处理PDF文档并构建向量数据库，然后进入交互式问答界面。

### 4. 使用示例

运行示例脚本查看系统功能：

```bash
python example.py
```

## 使用说明

1. **文档处理**：系统会自动处理项目中的PDF文档，提取文本并分割成适合检索的片段。

2. **向量数据库**：首次运行会构建向量数据库并保存，后续运行会直接加载已有的数据库。

3. **智能问答**：在交互式界面中输入问题，系统会基于向量检索和LLM生成专业的回答。

4. **专业领域**：系统专注于电力设备监控领域，提供专业、准确的回答。

## 技术栈

- **Langchain**：用于构建LLM应用和RAG系统
- **FAISS**：高效的向量搜索库
- **OpenAI API**：提供语言模型能力
- **PyPDF**：PDF文档处理
- **Python**：主要开发语言

## 扩展功能

系统设计具有良好的扩展性，可以通过以下方式增强功能：

1. **添加更多文档**：将更多电力设备监控相关的文档放入项目目录
2. **扩展工具集**：添加更多电力设备监控专用工具
3. **优化检索策略**：调整向量检索参数，提高检索准确性
4. **集成实时数据**：连接电力设备监控系统，获取实时数据

## 注意事项

- 需要有效的OpenAI API密钥才能使用系统
- 首次运行会构建向量数据库，可能需要较长时间
- 系统会自动保存向量数据库，后续运行速度会更快
- 建议使用Python 3.8+版本
